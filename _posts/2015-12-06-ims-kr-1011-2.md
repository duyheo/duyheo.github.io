<h1><a name="_Toc452399756">1 적용범위</a></h1>
우리나라 초∙중등 교육과정과 핵심성취 기준 문서 스킴을 기반으로 한 이 표준의 주요 목적은 문서로 발행된 교육 분야의 성취 기준을 기계적으로 해석하기 위해 필요한 요소 정의와 속성을 명시하는 것으로서 데이터 요소를 식별하고 해당 속성을 명시하는데 필요한 규칙을 포함한다.

이 표준은 교육 분야의 성취 기준을 링크드 데이터로 표현하기 위해 KS X 7004-1 프레임워크에서 정의한 템플릿과 용어를 적용하여 엔티티와 속성들을 정의한다. 이 응용 프로파일을 활용하는 교육 분야의 범위는 특별히 제한되지 않지만, 이 표준의 기반이 된 ASN Framework와 마찬가지로 우선 초∙중등 교육분야에 적용할 것을 권장한다.
<ul>
	<li>이 표준은 교육과정과 성취 기준 문서 스킴 비교 및 링크드데이터 클래스 설계를 위해 미국의 Achievement Standards Network(ASN) Framework를 참조하였다.</li>
</ul>

<h1><a name="_Toc439958783">3 용어와 정의</a></h1>

&nbsp;
<h2><a name="_Toc4399587831">3.1 링크드데이터</a></h2>
유일 자원 식별자(URI)로 식별되고, 각 유일 자원 식별자에 링크 정보를 부여함으로써 상호 연결된 데이터
&nbsp;
<h2><a name="_Toc4399587832">3.2 성취기준(achievement standards)</a></h2>
교수·학습 및 평가를 위한 실질적인 근거로서 각 교과목에서 학생들이 학습을 통해 성취해야 할 지식, 기능, 태도의 특성, 능력을 기술한 문서
&nbsp;
<h2><a name="_Toc4399587833">3.3 성취수준(achievement level)</a></h2>
학생들이 교과목별 성취 기준에 도달한 수준을 나타내는 것으로 성취 기준에 도달한 정도를 몇 개의 수준으로 구분하고 각 수준별 지식, 기능, 태도의 특성을 설명한 것

<strong>비고 1</strong> 성취 기준 단위의 성취수준은 ‘상-중-하’로 구분된다.

<strong>비고 2</strong> 학기 단위의 성취수준은 ‘A-B-C-D-E’ 등으로 구분된다.
&nbsp;
<h2><a name="_Toc4399587834">3.4 식별자(identifier)</a></h2>
개체를 고유하게 식별할 수 있는 문자열

<strong>비고 1</strong> 한 개의 식별자는 문자적으로 중립적이고, 번역이 되지 않는다.

<strong>비고 2</strong> 한 개의 식별자는 구성식별자들의 성향을 나타탤 수 있다. 즉, 고유식별자, 두 개 이상의 식별자들 및 데이터 요소들과 내부적으로 작용하는 것들이 규칙에 기반하고 함게 이루어져 “단일”식별자로 활용된다.

[ISO/IEC 11179-3:2003, 3.2.17]

<h1><a name="_Toc439958784">4 성취 기준 링크드 데이터 클래스 정의</a></h1>
성취 기준 링크드 데이터는 아래와 같이 3가지 클래스들로 구성되며, 이 중 핵심 클래스인 <em>Curriculum Document</em>와 <em>Achievement Statement</em> 사이의 관계 정의는 그림 1과 같이 표현할 수 있다.
<ul>
	<li>CurruculumStandard : 교육과정</li>
	<li>AchievementStatement : 교육과정의 성취 기준 또는 핵심 성취 기준</li>
	<li>LearningResource : 성취 기준과 연관된 학습자원</li>
</ul>

<h1><a name="_Toc439958788">5 성취 기준 링크드 데이터 속성 정의를 위한 네임스페이스</a></h1>
성취 기준 링크드 데이터 표현을 위한 <em>Curriculum Document</em>와 <em>Achievement Statement</em>  같은 클래스의 속성을 정의하기 위해 다음과 같은 네임스페이스를 사용한다.
<ul>
	<li>ASN (Achievement Standards Network) 어휘 (<a href="http://purl.org/ASN/schema/core/">http://purl.org/ASN/schema/core/</a>)</li>
	<li>DCMI 메타데이터 어휘 (<a href="http://purl.org/dc/terms/">http://purl.org/dc/terms/</a>)</li>
	<li>FOAF (Friend of a Friend) (<a href="http://xmlns.com/foaf/0.1/">http://xmlns.com/foaf/0.1/</a>)</li>
	<li>OWL (Web Ontology Language) (<a href="http://www.w3.org/2002/07/owl">http://www.w3.org/2002/07/owl#</a>)</li>
	<li>SKOS (Simple Knowledge Organization System) (<a href="http://www.w3.org/2004/02/skos/">http://www.w3.org/2004/02/skos/</a>)</li>
</ul>
이 표준이 참조한 ASN Framework로부터 수정 또는 재정의하지 않고 그대로 수용한 속성에 대해서는 ASN Framework의 속성 정보를 원문과 동일하게 적용한다.

<h1><a name="_Toc439958789">6 성취 기준 링크드 데이터 엔티티/속성 테이블</a></h1>
성취 기준 링크드 데이터의 클래스와 속성은 <strong>그림 2</strong>와 같은 관계로 표현될 수 있다.

<a href="{{ "/assets/images/ldaf_6.1.png.jpg" | absolute_url }}"><center><img class="aligncenter wp-image-1591 size-full" src="{{ "/assets/images/ldaf_6.1.png.jpg" | absolute_url }}" alt="ldaf_6.1.png" width="598" height="870" /></center></a>
<p style="text-align: center;">그림 2 — 성취 기준 링크드 데이터 클래스들의 속성 정의</p>
성취 기준 링크드 데이터 클래스 중에서 <em>LearningResource</em> 클래스는 학습 자원을 설명하는 메타데이터 형식을 정의하지 않으며, 세부적인 성취 기준을 설명하는 <em>AchievementStatement</em> 클래스와 학습 자원 간의 관계만을 정의한다. <em>LearningResource</em> 클래스는 어떤 유형의 학습 자원 메타데이터 형식과도 연결될 수 있는 일반 클래스이므로 학습 자원을 기술할 때는 <em>LearningResource</em> 클래스를 확장(extension)하여 사용할 것을 권장한다.
<ul>
	<li>디지털 자원을 위한 메타데이터 표준으로는 Dublin Core, Learning Resource Metadata Initiative (LRMI), Learning Object Metadata (LOM, IEEE 1484.12.1), Metadata for Learning Resources (MLR, ISO/IEC 19788 / KS X 7001) 등이 사용될 수 있다.</li>
</ul>
<ul>
	<li><em>LearningResource</em> 클래스를 학습 자원 메타데이터 형식으로 확장하여 사용하는 방법의 예시는 다음과 같다.</li>

<h2><a name="_Toc439958850">9.2 역량과 역량 간 매핑</a></h2>
역량과 역량 간 매핑은 초중등 교육과정에는 적용되지 않지만 고등교육과정 또는 그 밖의 교육과정에서 정의한 역량 간의 관계를 설정할 때 참고하기 위한 것이다.
